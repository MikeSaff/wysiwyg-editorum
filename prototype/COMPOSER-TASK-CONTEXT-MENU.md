# COMPOSER TASK: Контекстное меню для WYSIWYG-редактора

## Контекст
Файлы: `src/main.js`, `src/styles.css`, `src/toolbar.js`
Редактор: ProseMirror-based WYSIWYG в `index.html`
Стек: Vanilla JS + ProseMirror + Vite (ES modules)

## Задача
Создать контекстное меню, которое появляется:
1. **При правом клике** на элементах в редакторе (текст, формула, рисунок, таблица)
2. **При выделении текста** — floating toolbar (как в Medium/Notion)

## Требования

### Контекстное меню (правый клик)
Заменяет стандартное браузерное меню внутри `.ProseMirror`.

**На тексте:**
- Вырезать / Копировать / Вставить
- Стиль абзаца: Обычный / Заголовок 1-3 / Подпись рисунка / Подпись таблицы
- Выравнивание: Лево / Центр / Право / По ширине

**На формуле (.math-block, .math-inline):**
- Редактировать формулу (открывает prompt с LaTeX)
- Удалить формулу
- Копировать LaTeX

**На рисунке (.inline-image):**
- Увеличить (lightbox)
- Удалить
- Выравнивание: Лево / Центр / Право

**На таблице (table):**
- Добавить строку / столбец
- Удалить строку / столбец
- Объединить / разделить ячейки
- Удалить таблицу

### Floating toolbar (при выделении текста)
Появляется над выделенным текстом при mouseup.

**Кнопки:**
- B I U S (bold, italic, underline, strikethrough)
- Ссылка
- Формула (inline)

### Стилизация
- Фон: белый, тень, border-radius: 6px
- Шрифт: PT Sans, 13px
- Иконки: текстовые (как в toolbar)
- Анимация появления: fade-in 0.15s
- Позиционирование: абсолютное, рядом с курсором/выделением
- Закрытие: клик вне меню или Escape

### Реализация
Создать НОВЫЙ файл `src/context-menu.js` с экспортируемой функцией:
```js
export function setupContextMenu(view, editorEl) { ... }
```

Подключить в `main.js`:
```js
import { setupContextMenu } from "./context-menu.js"
// после создания view:
setupContextMenu(view, editorEl)
```

CSS добавить в `src/styles.css` в секцию `/* === Context Menu === */`

### Важно
- НЕ трогать `word-import.js` и `schema.js` — с ними работает Codex
- Использовать ProseMirror API для действий (toggleMark, setBlockType и т.д.)
- Импортировать schema из `./schema.js`
- Проверить: `npx vite build` должен пройти без ошибок
