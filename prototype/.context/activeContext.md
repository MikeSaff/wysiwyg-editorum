# Active Context — WYSIWYG Editorum

Единая точка правды: **кто за что отвечает**, **какие задачи активны**, **как фиксировать изменения**.

---

## Ticket protocol (обязательно)

После **каждого** осмысленного изменения кода или контрактов:

1. Добавить запись в **`.context/progress.md`** (таблица сессии или секция «История»): что сделано, файлы, агент/роль, статус.
2. Выполнить проверку билда: **`npx vite build`** (из каталога `prototype`) — должно завершаться без ошибок.
3. Коммиты в git выполняет **человек / назначенный агент** по правилам репозитория; автоматические агенты не коммитят без явного запроса.
4. Если билд упал — **сначала** исправление, **потом** запись в `progress.md` со статусом ✅ и пометкой про билд.

Кратко: **изменение → progress.md → билд OK**.

---

## Правила для агентов

- Не менять файлы **вне своей зоны владения** без согласования (см. таблицу ниже и запись в `progress.md`).
- Не трогать **`src/schema.js`** и **`src/word-import.js`** без задачи владельца (Claude / Codex) — см. ownership.
- Новые крупные фичи — тикет в `progress.md` (backlog / в работе) до или сразу после первого коммита изменений.

## File ownership (владение файлами)

| Файл / область | Владелец | Комментарий |
|-----------------|----------|-------------|
| `src/schema.js` | Claude | типы узлов, marks, атрибуты |
| `src/word-import.js` | Codex | OMML, таблицы, формулы, импорт .docx |
| `src/context-menu.js` | Composer | контекстное меню, floating toolbar |
| `src/typography-rules.js` | Composer | ГОСТ §16 input rules: кавычки, тире (импорт в `main.js`) |
| `src/toolbar.js` | Claude | кнопки панели инструментов |
| `src/main.js` | Claude | инициализация view, lightbox, навигация, autosave; Composer — только точечное подключение своих модулей (например `setupContextMenu`) |
| `src/styles.css` | Claude | общий CSS; Composer добавляет секции по согласованию (например `/* === Context Menu === */`) |
| `index.html` | Claude | разметка страницы |
| `COMPOSER-TASK-*.md` (корень) | Composer | постановка задач под Composer |
| `.context/*.md` | Общий | правила и трекер; обновляет тот, кто делает изменение (по Ticket protocol) |

**Конфликт зон:** нужно изменение чужого файла → запись в `progress.md` (запрос / согласование) или отдельный тикет владельцу.
